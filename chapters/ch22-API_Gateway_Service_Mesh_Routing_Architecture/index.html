    ‚≠ê Why Do We Need BOTH?¶

    üö™ API Gateway = External Door¶

    Controls traffic coming from the internet into the cluster.

    Provides:

    • Rate limiting
    • Authentication
    • API keys
    • Routing
    • WAF (security firewall)
    • TLS termination
    • Logging

    Popular gateways:

    • Kong
    • NGINX Ingress
    • Istio Gateway
    • Ambassador
    • Traefik

    üîÄ Service Mesh = Internal Smart Network¶

    Controls traffic inside the cluster between services.

    Provides:

    • mTLS encryption
    • Traffic splitting
    • Retries, timeouts
    • Observability
    • Zero-trust networking
    • Cross-service routing
    • Canary rollouts
    • Circuit breakers

    Most popular:

    • Istio
    • Linkerd
    • Consul

    [ Internet ]

    üß± PART 4 ‚Äî API Gateway (External Entry)¶

    You have two options:

    OPTION A ‚Äî Istio Gateway (built-in)¶

    We‚Äôll use Istio Gateway since you already have Istio.

    Step 1: Create Gateway¶

    gateway.yaml

    apiVersion: networking.istio.io/v1alpha3
kind: Gateway
metadata:
  name: app-gateway
  namespace: prod
spec:
  selector:
    istio: ingressgateway
  servers:
    - port:
        number: 80
        name: http
        protocol: HTTP
      hosts:
        - "*"

    üß± PART 5 ‚Äî Route Internet ‚Üí Frontend ‚Üí Backend¶

    virtualservice.yaml

    apiVersion: networking.istio.io/v1alpha3
kind: VirtualService
metadata:
  name: app-route
  namespace: prod
spec:
  gateways:
    - app-gateway
  hosts:
    - "*"
  http:
    - match:
        - uri:
            prefix: "/api"
      route:
        - destination:
            host: backend
            port:
              number: 3000
    - route:
        - destination:
            host: frontend
            port:
              number: 80

    ‚úîÔ∏è /api ‚Üí backend ‚úîÔ∏è everything else ‚Üí frontend

    This is how websites are built in real companies.

    üîê PART 6 ‚Äî Add TLS (HTTPS)¶

    Add TLS to your gateway:

  - port:
      number: 443
      name: https
      protocol: HTTPS
    hosts:
      - "shop.example.com"
    tls:
      mode: SIMPLE
      credentialName: tls-secret

    Provide cert via:

    kubectl create secret tls tls-secret --key privkey.pem --cert cert.pem -n prod

    ‚úîÔ∏è Production-grade HTTPS with Istio Gateway.

    üõ°Ô∏è PART 7 ‚Äî Apply Zero-Trust Security (mTLS)¶

    Force all Pods in mesh to require encryption:

    apiVersion: security.istio.io/v1beta1
kind: PeerAuthentication
metadata:
  name: default
  namespace: prod
spec:
  mtls:
    mode: STRICT

    Now:

    ‚úîÔ∏è frontend ‚Üî backend is encrypted ‚úîÔ∏è No service can talk unless allowed

    üõ°Ô∏è PART 8 ‚Äî Apply Service-to-Service Authorization (RBAC)¶

    Allow frontend to talk to backend:

    apiVersion: security.istio.io/v1beta1
kind: AuthorizationPolicy
metadata:
  name: allow-frontend
  namespace: prod
spec:
  selector:
    matchLabels:
      app: backend
  rules:
    - from:
        - source:
            principals: ["cluster.local/ns/prod/sa/default"]

    Everything else is blocked.
