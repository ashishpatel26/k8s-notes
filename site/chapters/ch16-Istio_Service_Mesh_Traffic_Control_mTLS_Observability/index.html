    ‚≠ê What Is a Service Mesh?¶

    Think of a service mesh as:

    ‚ÄúA smart traffic controller that sits between all your services.‚Äù

    It uses sidecar proxies (Envoy) injected into every Pod.

    These sidecars handle:

    • security
    • routing
    • metrics
    • logging
    • retries
    • service-to-service encryption

    Your code doesn‚Äôt have to change. Istio does the heavy lifting.

    üß± Step 1 ‚Äî Install Istio (Easy Way)¶

    Download Istio CLI:

    curl -L https://istio.io/downloadIstio | sh -

    Move into Istio folder:

    cd istio-1.*/

    Install ‚Äúdemo‚Äù profile (perfect for learning):

    istioctl install --set profile=demo -y

    Check Istio components:

    kubectl get pods -n istio-system

    You‚Äôll see:

    • istiod
    • istio-ingressgateway

    ‚úîÔ∏è Mesh installed!

    üß© Step 2 ‚Äî Label Your Namespace to Enable Sidecars¶

    Pick the namespace you want to mesh (example: dev):

    kubectl label namespace dev istio-injection=enabled

    Now every new Pod in dev will automatically inject an Envoy sidecar:

    app-container + envoy-proxy

    Check with:

    kubectl get pods -n dev

    Each Pod should have 2 containers.

    ‚ö° Step 6 ‚Äî Istio VirtualService (traffic splitting)¶

    Create:

    backend-traffic.yaml

    apiVersion: networking.istio.io/v1alpha3
kind: VirtualService
metadata:
  name: backend
  namespace: dev
spec:
  hosts:
    - backend
  http:
    - route:
        - destination:
            host: backend
            subset: v1
          weight: 90
        - destination:
            host: backend
            subset: v2
          weight: 10

    And:

    destination-rule.yaml

    apiVersion: networking.istio.io/v1alpha3
kind: DestinationRule
metadata:
  name: backend
  namespace: dev
spec:
  host: backend
  subsets:
    - name: v1
      labels:
        version: v1
    - name: v2
      labels:
        version: v2

    Apply:

    kubectl apply -f destination-rule.yaml -f backend-traffic.yaml

    üéâ Congratulations:

    Your traffic is now:

    • 90% ‚Üí backend v1

    This is canary deployment, used by Google & Netflix.
